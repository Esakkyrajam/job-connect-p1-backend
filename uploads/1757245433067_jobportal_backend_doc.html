Job Portal Backend Documentation

This document provides the backend structure, explanations, API endpoints, and dummy data examples for testing in Postman.

üìÇ Backend Structure

  • entity: User, Recruiter, Job, Application
  • repository: JPA Repositories for database access
  • service: Business logic for authentication, job management, applications
  • controller: REST APIs (AuthController, JobController, ApplicationController, FileController)
  • config: Security and JWT configuration

üîë Authentication Endpoints

1. Register User

POST http://localhost:8080/api/auth/signup

Dummy Data:
{ "username": "raj", "email": "raj@example.com", "password": "password123", "role": "JOBSEEKER" }

2. Login

POST http://localhost:8080/api/auth/login

Dummy Data:
{ "email": "raj@example.com", "password": "password123" }

üíº Job Endpoints

1. Post Job (Recruiter only)

POST http://localhost:8080/api/jobs

Dummy Data:
{ "title": "Java Developer", "description": "Spring Boot developer needed", "location": "Chennai", "salary": 600000 }

2. List All Jobs

GET http://localhost:8080/api/jobs

3. Search Jobs

GET http://localhost:8080/api/jobs/search?title=Java&location=Chennai

üìÑ Application Endpoints

1. Apply to Job

POST http://localhost:8080/api/applications/{jobId}/{jobseekerId}

Dummy Data:
POST http://localhost:8080/api/applications/1/2

2. Get Jobseeker Applications

GET http://localhost:8080/api/applications/jobseeker/{jobseekerId}

3. Get Job Applications (Recruiter)

GET http://localhost:8080/api/applications/job/{jobId}

üìÇ File Upload

1. Upload Resume

POST http://localhost:8080/api/files/upload

Send as form-data with key file
